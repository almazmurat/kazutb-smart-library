# REST API Documentation — KazUTB Smart Library

## Base URL

```
/api/v1/
```

All endpoints are versioned. Current version: `v1`.

## Authentication

Protected endpoints require a Bearer token in the `Authorization` header:

```
Authorization: Bearer <access_token>
```

Tokens are obtained via `POST /api/v1/auth/login`.

## Response Format

### Success

```json
{
  "data": { ... },
  "meta": { "page": 1, "total": 150, "limit": 20 }
}
```

### Error

```json
{
  "statusCode": 400,
  "error": "Bad Request",
  "message": "Validation failed",
  "details": [{ "field": "isbn", "message": "isbn must be a valid ISBN" }],
  "timestamp": "2026-03-17T10:00:00Z",
  "path": "/api/v1/books"
}
```

## Pagination

List endpoints support:

- `?page=1` — page number (1-based)
- `?limit=20` — items per page (max 100)
- `?sortBy=title` — sort field
- `?sortOrder=asc|desc` — sort direction

## Filtering

Book list supports:

- `?title=algebra` — filter by title fragment
- `?authorId=<uuid>` — filter by author
- `?categoryId=uuid` — filter by category
- `?branchId=<uuid>` — filter by owning branch
- `?isActive=true|false` — filter by active flag

Book copy management endpoints:

- `POST /api/v1/books/:bookId/copies`
- `GET /api/v1/books/:bookId/copies`
- `PATCH /api/v1/books/:bookId/copies/:copyId`

## Search

GET /api/v1/search?q=алгебра&lang=ru&year=2020&page=1
GET /api/v1/search/autocomplete?q=алг
GET /api/v1/search/similar/:bookId
