# ROLES_AND_ACCESS
> Derived from [[PROJECT_CONTEXT]] §4

## Role definitions
| Role | Who | Scope |
|---|---|---|
| Guest | Anonymous or non-AD visitor | Public read-only access |
| Student | AD-authenticated student | Full member access |
| Teacher | AD-authenticated teacher | Full member access |
| Employee | AD-authenticated non-library staff | Full member access |
| Librarian | Library staff | Full operational workflow |
| Admin | System administrator | Full system control |

## Ordinary users
Student, teacher, and employee share the same functional scope and are treated as one member group.

## Guest access
Guests can browse the catalog, full metadata, availability, locations, external resources, news, homepage sections, and scientific-work metadata — but cannot reserve, read protected content, or use the dashboard.

## Staff access
Librarians operate cataloging, circulation, moderation, and reporting. Admins add user management, settings, integrations, full logs, and system-wide governance.
