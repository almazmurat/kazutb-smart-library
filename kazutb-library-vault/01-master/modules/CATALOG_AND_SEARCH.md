# CATALOG_AND_SEARCH
> Derived from [[PROJECT_CONTEXT]] §7, §8, and §29

## Purpose
The catalog is the core user-facing entry point to the library fund. It must support real discovery, not decorative listing.

## Required book card fields
Every book card should show title, author, publisher, year, language, UDC, author sign, availability, copy counts, branch, sigla, and precise shelf location.

## Search capabilities
| Capability | Status |
|---|---|
| Title, author, keyword, metadata search | Required now |
| Full-text search and filters | Required now |
| UDC and branch filtering | Required now |
| Autocomplete and AI search | Future |

## UX requirements
Filters must be fast, persistent during the session, and visually clear. UDC and author sign should remain visible on the cover UI because they are part of real library practice.
