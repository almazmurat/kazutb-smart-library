# NOTIFICATIONS
> Derived from [[PROJECT_CONTEXT]] §25

## Channels
- in-app notifications
- email notifications through the user's corporate address

## Event coverage
Notifications should cover reservation state changes, due-soon and overdue reminders, renewals, digital access grants, scientific-work moderation events, contact-message handling, and optionally news publication.

## User controls
Users can view all notifications in the dashboard and mark them as read. Admin can configure which events also send email.
