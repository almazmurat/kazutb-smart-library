## Key Decisions Made

### Decision 1: Two-Level Instruction System
- **Rule Level**: `.instructions.md` — What agent MUST do (high-level)
- **Pattern Level**: `AGENT_TRACKING_INSTRUCTIONS.md` — HOW to do it per task type (detail level)
- **Rationale**: Separation allows rules to evolve independently from patterns
- **Cross-Link**: [[decision-instruction-architecture-2026-04-15]]

### Decision 2: Auto-Sync at Session End (Not Continuous)
- **Approach**: Single `bash scripts/dev/auto-sync.sh` call at session end
- **NOT**: Continuous polling or background daemon (would complicate Docker)
- **Rationale**: Agent explicitly calls it when done, faster than monitoring
- **Accepted Trade-off**: 2-3 minute window of uncommitted code (acceptable because agent is synchronous)

### Decision 3: Memory Fragments as Source of Truth
- **Implementation**: Every code change → memory entry → git commit → auto-sync
- **NOT**: Auto-extract from git diffs (lossy, loses "why")
- **Rationale**: Memory explains intent, git shows what; both needed
- **Integration**: WORK_LOG acts as daily micro-index
